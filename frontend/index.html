Contiguous Allocation — Memory Layout

Sequential Blocks

Allocation Table

Sr. NoTitleStart BlockLengthEnd Block

How it works

Each book occupies a contiguous (consecutive) set of disk blocks. The directory stores only the start block and the length.

  • Fast sequential access
  • Simple directory structure
  • External fragmentation over time
  • File size must be known in advance

Linked Allocation — Block Chain

Pointer-based

Directory Table

Sr. NoTitleStart BlockEnd BlockBlocks

How it works

Each block holds data and a pointer to the next block. Blocks can be scattered anywhere on disk — no contiguity required.

  • No external fragmentation
  • Dynamic file growth
  • Sequential access only
  • Pointer overhead per block
  • Poor random access time

Indexed Allocation — Index Blocks

Index Table

Directory Table

Sr. NoTitleIndex BlockData BlocksPointers

How it works

An index block stores all block pointers for a file. The directory just points to the index block, enabling direct access to any data block.

  • Supports direct access
  • No external fragmentation
  • Dynamic growth
  • Index block overhead
  • Wasteful for small files

Contiguous Allocation

Method 01

Files are stored in consecutive disk blocks. The OS only needs to know where a file starts and how many blocks it uses.

Directory entry: (start, length)

Analogy

Like shelving books in a row on continuous shelf space. Easy to find, but you need to reserve the whole shelf in advance.

Linked Allocation

Method 02

Each disk block contains data and a pointer to the next block. Blocks for a single file can be dispersed throughout the disk.

Block: [data | next_ptr]

Analogy

Like a treasure hunt — each shelf tag tells you where the next book chapter is hidden.

Indexed Allocation

Method 03

A dedicated index block holds pointers to all data blocks of a file. The directory entry points only to this index block.

Index block: [ptr0, ptr1, …, ptrN]

Analogy

Like a book's table of contents — one page lists where every chapter is, so you can jump directly to any section.
